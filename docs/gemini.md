# Lattara Website Overhaul Context

## Project Overview
We are overhauling the Lattara (MarTech/RevOps consulting) website to move away from a generic AI-generated look. The new aesthetic is **"The Data Architect."**

## Design System & Aesthetic
- **Vibe:** Engineering-grade, precise, technical, "blueprint" style.
- **Typography:**
  - **Headings/UI Elements:** `JetBrains Mono` (Monospace)
  - **Body Text:** `Inter` (Sans-serif)
- **Colors:**
  - `brand-red`: `#9D2235`
  - `brand-dark`: `#0F172A` (High contrast slate)
  - `brand-gray`: `#F1F5F9` (Technical background)
- **Visual Elements:**
  - Grid backgrounds (`bg-grid-pattern`).
  - Sharp corners (no rounded-lg).
  - Visible borders and "module" styling.
  - Terminal/Code block aesthetics.
- **Tone of Voice:**
  - Use terms like: "Protocols", "Architecture", "Deployment", "Telemetry", "Throughput", "Latency", "System Status".
  - Avoid marketing fluff. Be direct and concise.

## Development Protocols
- **Version Control:** The Gemini agent is authorized to commit any changes directly to the GitHub repository upon completion of a task. Always use concise, semantic commit messages.
